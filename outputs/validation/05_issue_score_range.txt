================================================================================
ISSUE SIMILARITY WEIGHT RANGE PROOF
================================================================================

── Mathematical Foundation ──

Bray-Curtis per issue k:
  BC_k(i,j) = 1 - |f_i,k - f_j,k| / (f_i,k + f_j,k)
  where f_i,k = firm i's spend on issue k / firm i's total budget

Property: BC_k ∈ [0, 1] for all k (bounded)
  Proof: |f_i,k - f_j,k| ≤ f_i,k + f_j,k always, so BC ≥ 0.
         BC = 1 - |...|/(sum) ≤ 1.

── Sum of Bounded Values ──

Let N = number of shared issues between firms i and j.
Sum of BC scores:
  S(i,j) = sum over shared issues k: BC_k(i,j)

Since each BC_k ∈ [0, 1]:
  S(i,j) ≤ N × 1.0 = N

Example with N = 75 shared issues (all 75 issue codes):
  Maximum sum S = 75 (if BC = 1.0 on each issue)
  Minimum sum S = 0  (if BC = 0 on all issues)

── Impact of Normalization (sqrt) ──

With NORMALIZE=True:
  weight(i,j) = S(i,j) / sqrt(N)

For N shared issues with perfect BC = 1.0 on each:
  weight_max(N) = N / sqrt(N) = sqrt(N)

  Synthetic examples (perfect alignment on N issues):
    N =  1 issues → max weight =  1 / sqrt( 1) = 1.0000
    N =  4 issues → max weight =  4 / sqrt( 4) = 2.0000
    N =  9 issues → max weight =  9 / sqrt( 9) = 3.0000
    N = 25 issues → max weight = 25 / sqrt(25) = 5.0000
    N = 49 issues → max weight = 49 / sqrt(49) = 7.0000
    N = 75 issues → max weight = 75 / sqrt(75) = 8.6603

  Theoretical maximum (N = 75): 8.6603

With NORMALIZE=False:
  weight(i,j) = S(i,j) directly (no sqrt divisor)
  Theoretical maximum: 75.0 (if all 75 issues have BC = 1.0)

── Why Unbounded Weights Are Intentional ──

Design rationale: three normalization choices and their tradeoffs:

1. Raw sum (NORMALIZE=False, weight = N × mean_BC):
   - Rewards breadth linearly: pairs with 75 shared issues get
     much heavier weights than pairs with 10 shared issues.
   - Unbounded: up to 75.
   - Pro: captures full breadth signal (many shared issues)
   - Con: weight dominance driven entirely by breadth, not quality.

2. Plain mean (weight = mean_BC, bounded [0, 1]):
   - Removes breadth signal entirely: a pair with 1 issue at BC=0.9
     gets the same weight as a pair with 75 issues at BC=0.9.
   - Bounded in [0, 1].
   - Pro: weights are interpretable as similarity scores.
   - Con: ignores multiplicity of coordination across issues.

3. sqrt normalization (NORMALIZE=True, weight = sum_BC / sqrt(N)):
   - Compromise: rewards breadth SUB-LINEARLY.
   - Maximum bounded by sqrt(N), e.g., sqrt(75) ≈ 8.66.
   - Pro: balances depth (portfolio alignment) with breadth.
         Pairs coordinating on many issues get higher weights,
         but growth is sublinear (diminishing returns per issue).
   - Con: weights exceed [0, 1], cannot be interpreted as fractions.

CHOSEN: Option 3 (sqrt normalization).
This is best for community detection: algorithms like Leiden
handle arbitrary positive weights without assuming [0, 1].

── Interpretation of Unbounded Weights ──

Example: weight(A, B) = 3.0

Do NOT interpret as: A and B have 3.0 units of similarity.
DO interpret as:
  - A and B are aligned on multiple issues.
  - If they share 9 issues with perfect BC on each: 9/sqrt(9) = 3.
  - If they share 4 issues with perfect BC on each: 4/sqrt(4) = 2.
  - The weight reflects both depth (per-issue alignment)
    and breadth (number of shared issues).

For centrality and community detection, this is appropriate:
  - Eigenvector centrality: algorithms designed for weighted graphs
    with arbitrary edge weights. [0, 1] not required.
  - Leiden modularity: likewise, works with positive weights.
    Unbounded weights just amplify within-community vs.
    between-community edge strength more aggressively.

── Summary ──

Issue similarity weights are NOT bounded in [0, 1] because:

1. We sum Bray-Curtis scores across multiple shared issues.
   Sum of N values in [0,1] can be at most N.

2. We normalize by sqrt(N), not by N itself.
   This rewards breadth (many shared issues) sub-linearly.

3. With 75 issue codes, theoretical max = sqrt(75) ≈ 8.66.
   Without normalization, max = 75.

4. This is intentional: it balances depth (per-issue alignment)
   with breadth (strategic coordination across multiple areas).

5. Users should NOT interpret weights as [0,1] similarity scores.
   Instead: a weight of 3.0 roughly means ~9 shared issues with
   perfect alignment. Community detection algorithms handle this.

See design_decisions.md §11 for the design rationale.
